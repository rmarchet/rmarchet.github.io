üìã API Overview

The User Management API provides a comprehensive set of endpoints for managing user accounts, authentication, and user profiles. This RESTful API follows standard HTTP conventions and returns JSON responses.

Base URL

https://api.userservice.com/v1

Response Format

All API responses are in JSON format and include standard HTTP status codes. Error responses include detailed error messages and codes for debugging.

üí° Note: For demonstration purposes, we're using httpbin.org endpoints which echo back the request data. In a real implementation, these would be your actual API endpoints.

üîê Authentication

The API uses Bearer token authentication. Include your API token in the Authorization header for all requests.

POST /auth/login

Authenticate a user and receive an access token.

Request Body
email
string
User's email address
password
string
User's password

Example Response

200OK { "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "token_type": "Bearer", "expires_in": 3600, "user": { "id": 1, "email": "john@example.com", "name": "John Doe" } }

üë• User Management

GET /users

Retrieve a list of all users with optional filtering and pagination.

Query Parameters
page
integer
Page number for pagination (default: 1)
limit
integer
Number of users per page (default: 20, max: 100)
role
string
Filter by user role (admin, user, moderator)
GET /users/{id}

Get detailed information about a specific user by their ID.

Path Parameters
id
integer
Unique identifier for the user

Example Response

200OK { "id": 1, "email": "john@example.com", "name": "John Doe", "role": "user", "created_at": "2023-01-15T10:30:00Z", "last_login": "2023-12-08T14:22:00Z", "profile": { "avatar": "https://example.com/avatars/john.jpg", "bio": "Software developer passionate about APIs", "location": "San Francisco, CA" } }

POST /users

Create a new user account.

Request Body
email
string
User's email address (must be unique)
name
string
User's full name
password
string
User's password (min 8 characters)
role
string
User role (optional, default: "user")
PUT /users/{id}

Update an existing user's information.

DELETE /users/{id}

Delete a user account permanently.

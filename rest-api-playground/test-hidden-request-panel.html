🙈 Hidden Request Panel Mode

Test the new hideRequestPanel configuration option

📋 What's New:

The hideRequestPanel: true option completely hides the Request panel and shows the request info in the Response panel header instead.

🔧 Standard Layout

  • Request panel: 50% width
  • Response panel: 50% width
  • Full request configuration UI

🙈 Hidden Request Panel

  • Request panel: Hidden completely
  • Response panel: 100% width
  • Compact request info in header

✨ Hidden Panel Features:

  • Full Width Response: Response panel takes up the entire space
  • Request Summary: Method badge, URL, and Send button in Response header
  • Color-coded Methods: GET (green), POST (blue), PUT (orange), DELETE (red), etc.
  • URL Truncation: Long URLs are truncated with ellipsis and show full URL on hover
  • Compact Send Button: Smaller button integrated into the header
  • Programmatic Control: Request can still be modified via API methods

🎮 Interactive Demo

The playground below has hideRequestPanel: true. Notice how the request info appears in the Response panel header!

💡 Request Info in Header:
Method Badge: Color-coded HTTP method
URL Display: Current request URL (truncated if long)
Send Button: Trigger the request directly from header

🎯 Use Cases:

  • Response Focus: Maximum space for analyzing API responses
  • Fixed Requests: When request configuration doesn't need to change
  • Embedded Widgets: Clean integration in dashboards or monitoring tools
  • Mobile/Small Screens: Better use of limited screen space
  • Read-only APIs: Focus on GET requests and response exploration

🔧 Configuration:

const demo = new RestAPIClient({
  config: {
    title: 'Response-Focused Playground',
    hideRequestPanel: true,
    defaultUrl: 'https://api.example.com/data'
  }
});

üöÄ API Overview

The Payment API enables you to accept payments, manage transactions, and handle refunds. Our REST API supports multiple payment methods including credit cards, digital wallets, and bank transfers.

Base URL

https://api.payments.com/v1

Response Format

All API responses are returned in JSON format. Successful requests return HTTP status codes in the 2xx range, while errors return 4xx or 5xx status codes with detailed error information.

üí° Demo Note
This documentation uses httpbin.org and JSONPlaceholder for demonstration purposes. In production, these would be your actual payment API endpoints.

üîê Authentication

The Payment API uses API keys for authentication. Include your secret API key in the Authorization header using Bearer authentication.

Authorization: Bearer sk_test_4eC39HqLyjWDarjtT1zdp7dc

API Keys

API keys are available in your dashboard. Use test keys for development and live keys for production.

Key Type Format Purpose
sk_test_... Secret Key Server-side development and testing
pk_test_... Publishable Key Client-side development and testing
sk_live_... Live Secret Key Production server-side operations

üí≥ Create Payment

POST /payments

Create a new payment with the specified amount and payment method.

Parameters

Parameter Type Required Description
amount integer required Amount in smallest currency unit (e.g., cents for USD)
currency string required Three-letter ISO currency code (e.g., USD, EUR)
payment_method string required Payment method ID or card token
description string optional Description of the payment
customer string optional Customer ID to associate with payment

Example Response

{ "id": "pi_1234567890", "object": "payment_intent", "amount": 2500, "currency": "usd", "status": "succeeded", "description": "Payment for Order #1234", "created": 1640995200, "customer": "cus_1234567890" }
